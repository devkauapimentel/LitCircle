## Metodologia resumida

```
SEGUNDA: Planning (15 min) — cada um pega 1 issue do board
DIÁRIO:  Mensagem no WhatsApp: "fiz X, vou fazer Y, travei em Z"
DOMINGO: Review (15 min) — mostra o que fez, planeja a próxima semana
```

**Regras:**
- WIP = 1 (só 1 issue por vez por pessoa)
- 45 min travado → documenta o problema → chama o parceiro
- Todo código via PR → parceiro revisa → merge
- Nunca commit direto na main

---

## Como aprender

1. Pega a issue
2. Lê 30 min da documentação oficial da tech que precisa
3. Tenta implementar
4. Travou → busca na documentação → tenta de novo
5. 45 min travado → documenta → chama parceiro
6. Funcionou → commit → PR → próxima issue

**Recursos de estudo estão no [12-guia-de-aprendizado.md](12-guia-de-aprendizado.md)**
